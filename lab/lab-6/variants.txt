Вариант 1:

Соединиться с сервером [Tokyo Tyrant](http://fallabs.com/tokyotyrant/) и записать значение "тест" по ключу "ключ". Проверить наличие значения с помощью родного консольного клиента `tcrtest`.

Вариант 2:

Соединиться с сервером [Memcached](http://memcached.org/) и записать, используя тестовый протокол, значение "тест" по ключу "ключ". Проверить наличие значения с помощью программы `telnet`.

Вариант 3:

Соединиться с сервером [Redis](http://redis.io/) и записать значение "тест" по ключу "ключ". Проверить наличие значения с помощью родного консольного клиента `redis-cli`.

Вариант 4:

Соединиться с сервером IRC [InspIRCd](http://www.inspircd.org/), создать канал "#test". Подключиться к каналу также другим IRC-клиентом (например, mIRC, Chatzilla...). После этого отправить из программы сообщение "тест", которое должен получить другой клиент.

Вариант 5:

Отправить сообщение на сервер [StatsD](https://github.com/etsy/statsd/) `тест:1|g` и убедиться в его получении через консоль сервера (сервер должен быть сконфигурирован с бекэндом Console).

Вариант 6:

Cоединиться с сервером [Apache ActiveMQ](http://activemq.apache.org/) по протоколу [STOMP](http://stomp.github.io/) и отправить сообщение "тест" в очередь "test". Проверить наличие сообщения родным консольным клиентом [activemq-admin](http://activemq.apache.org/activemq-command-line-tools-reference.html).

Вариант 7:

Создать в текущей директории текстовый файл "тест.txt" с содержимым "тест". Cоединиться с FTP-сервером [vsftpd](https://security.appspot.com/vsftpd.html) и загрузить этот файл  в папку /tmp/test файл. Проверить наличие файла командами `ls /tmp/test` и `cat /tmp/test/тест.txt`.

Вариант 8:

Cоединиться с SMTP-сервером [Postfix](http://www.postfix.org/) и отправить письмо для пользователя `root` с темой "тест" и содердимым "тест". Проверить получение письма утилитой `mailx`.

Вариант 9:

Соединиться с сервером [CollectD](https://collectd.org/) и отправить туда сообщение "тест" со значением "1179574444:123:456" и убедиться в его получении через CSV-файл (сервер должен быть настроен на запись в CSV).

Вариант 10:

Соединиться с сервером [Kyoto Tycoon](http://fallabs.com/kyototycoon/) по протоколу HTTP и записать значение "тест" по ключу "ключ". Проверить наличие значения с помощью родного консольного клиента `ktutiltest`.
